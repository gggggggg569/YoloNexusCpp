# YoloNexusCpp

YoloNexusCpp — приложение для Windows x64, которое захватывает изображение с монитора, окна или камеры, выполняет распознавание объектов моделью YOLOv8 в формате ONNX и показывает результат в отдельном окне. Инференс выполняется через ONNX Runtime и DirectML, поэтому может использовать совместимые видеокарты AMD, Intel и NVIDIA.

## Быстрый старт

### Требования

- Windows 10 или Windows 11 x64.
- Visual Studio с компонентом **Desktop development with C++** («Разработка классических приложений на C++»).
- CMake версии 3.25 или новее.
- vcpkg: команда `vcpkg` должна быть доступна в `PATH`, либо должна быть задана переменная окружения `VCPKG_ROOT`.
- Актуальный драйвер видеокарты с поддержкой DirectML.

Откройте PowerShell в корневой папке проекта.

### 1. Установка зависимостей

```powershell
vcpkg install --triplet x64-windows --x-wait-for-lock
```

Команда читает `vcpkg.json` и устанавливает OpenCV, ONNX Runtime DirectML и их зависимости в локальную папку `vcpkg_installed`.

### 2. Конфигурация CMake

```powershell
cmake --preset windows-x64
```

Команда использует preset `windows-x64` из `CMakePresets.json`, находит установленную Visual Studio, подключает vcpkg, включает тесты и создаёт каталог `build`.

Эту команду нужно выполнить один раз после получения чистой копии проекта. Повторная конфигурация нужна после удаления каталога `build` или существенного изменения CMake-файлов.

### 3. Сборка Release

```powershell
cmake --build --preset release --parallel
```

Команда собирает Release-версию в несколько потоков. Готовая программа находится здесь:

```text
build\Release\YoloNexusCpp.exe
```

Рядом с EXE автоматически копируются:

- `config.txt`;
- `best4k.onnx`;
- DLL OpenCV и связанных библиотек;
- `runtimes\directml\onnxruntime.dll`;
- `runtimes\directml\DirectML.dll`;
- остальные DLL, необходимые ONNX Runtime и DirectML.

### 4. Запуск тестов

```powershell
ctest --preset release
```

Команда запускает пять тестов Release-сборки. При успешном выполнении в конце появится `100% tests passed`.

### Сборка через Visual Studio

Можно открыть `YoloNexus.slnx`, выбрать `Release | x64` и выполнить **Build → Build Solution**. Перед этим зависимости также должны быть установлены командой vcpkg из корня проекта.

Результат сборки Visual Studio находится здесь:

```text
out\x64\Release\YoloNexusCpp.exe
```

## Запуск приложения

1. Убедитесь, что `config.txt` и модель `best4k.onnx` находятся рядом с `YoloNexusCpp.exe`. При штатной сборке они копируются автоматически.
2. Запустите `YoloNexusCpp.exe`.
3. Программа выведет список найденных мониторов, окон и камер.
4. Введите номер нужного источника и нажмите Enter.
5. После загрузки модели откроется окно предпросмотра с рамками обнаруженных объектов.

Управление:

- `Ctrl+C` — остановить приложение(не рекомендую);
- правый `Alt` — остановить приложение и оставить консоль открытой;
- закрытие окна предпросмотра — завершить конвейер обработки.

## Конфигурация

Файл `config.txt` читается из каталога, в котором находится EXE.

Текущий пример:

```text
model_path: ""

input_w: 736
input_h: 736

classes: full

draw_fps: 120
inference_fps: 120

out_w: 1280
out_h: 720
```

Назначение параметров:

- `model_path` — путь к ONNX-модели.
- `input_w`, `input_h` — размер входного тензора модели. Значения должны совпадать с размером, который ожидает модель.
- `classes: full` — использовать все классы из метаданных `names` внутри ONNX-модели.
- `classes: "0:person, 2:car"` — пример явного ограничения списка классов.
- `draw_fps` — целевая частота захвата и отрисовки.
- `inference_fps` — целевая частота инференса. Фактическое значение ограничивается `draw_fps`.
- `out_w`, `out_h` — начальный размер окна предпросмотра.

## Как работает приложение

1. `wmain` передаёт управление консольному приложению.
2. Загружается и проверяется `config.txt`.
3. Программа находит доступные мониторы, окна и камеры и предлагает выбрать источник.
4. Создаётся соответствующий модуль захвата.
5. Из папки `runtimes\directml` загружается ONNX Runtime с DirectML.
6. Модель YOLO инициализируется с указанным разрешением и списком классов.
7. Поток захвата постоянно публикует самый свежий кадр.
8. Поток инференса получает последний кадр, подготавливает NCHW-тензор, запускает модель и выполняет постобработку с NMS.
9. Основной runtime-поток масштабирует кадр, рисует рамки и подписи и передаёт изображение в Direct3D 11.
10. Раз в секунду обновляются FPS, задержки, загрузка CPU/GPU, память и статистика обнаружений.

Конвейер намеренно хранит только самый свежий кадр и последние результаты детекции. Если инференс работает медленнее захвата, старые кадры не накапливаются в очереди, благодаря чему уменьшается задержка предпросмотра.

## Назначение файлов

### Корень проекта

- `README.md` — документация, быстрый старт и описание архитектуры.
- `model.onnx` — ONNX-модель, используемая приложением по умолчанию.
- `config.txt` — параметры модели, FPS, разрешения и классов.
- `CMakeLists.txt` — описывает библиотеку ядра, приложение, тестовые исполняемые файлы, зависимости и копирование runtime-файлов.
- `CMakePresets.json` — содержит configure preset `windows-x64`, build preset `release` и test preset `release`.
- `vcpkg.json` — объявляет зависимости `opencv4` и локальный пакет `onnxruntime-directml`.
- `vcpkg-configuration.json` — указывает vcpkg использовать overlay-порты из папки `ports`.
- `YoloNexus.slnx` — переносимое XML-решение Visual Studio без абсолютных путей.
- `YoloNexus.vcxproj` — настройки сборки Debug/Release x64, список исходников, библиотеки и post-build копирование DLL/модели/конфигурации.
- `YoloNexus.vcxproj.filters` — группирует файлы проекта по папкам в обозревателе решений Visual Studio.

### `ports/onnxruntime-directml`

- `vcpkg.json` — manifest локального порта ONNX Runtime DirectML.
- `portfile.cmake` — загружает официальные NuGet-пакеты ONNX Runtime DirectML и Microsoft DirectML, устанавливает заголовки, `.lib` и DLL в дерево vcpkg.

### `src/app`

- `main.cpp` — точка входа `wmain`; запускает консольное приложение.
- `console_app.h` — объявление функции запуска консольного приложения.
- `console_app.cpp` — настройка UTF-8, COM и таймера Windows; чтение конфигурации; вывод источников; выбор источника пользователем; запуск и остановка runtime; обработка `Ctrl+C` и правого Alt.
- `console_dashboard.h` — интерфейс консольной панели телеметрии.
- `console_dashboard.cpp` — форматирует и обновляет в консоли FPS, задержки, загрузку ресурсов и статистику обнаружений.

### `src/application`

- `app_settings.h` — структура `AppSettings` со всеми параметрами запуска и объявления функций проверки настроек.
- `app_settings.cpp` — проверяет путь к модели, разрешения, FPS, классы и выбранный источник; вычисляет эффективный FPS инференса.
- `config_file.h` — объявление загрузчика `config.txt`.
- `config_file.cpp` — разбирает формат `key: value`, обязательные поля, классы и относительный путь модели; обнаруживает неизвестные и повторяющиеся параметры.
- `runtime_snapshot.h` — структуры состояния приложения и телеметрии, которые передаются консольной панели.
- `runtime_controller.h` — интерфейс контроллера жизненного цикла конвейера.
- `runtime_controller.cpp` — создаёт источник захвата, загружает ONNX Runtime и модель, запускает worker-потоки, управляет окном Direct3D, рисует кадры и обновляет телеметрию.

### `src/core`

- `types.h` — общие типы `Detection`, `DetectionClass` и `FramePacket`.
- `ema.h` — шаблон экспоненциального скользящего среднего для сглаживания измеряемых задержек.
- `timing.h` — объявление точного ожидания до заданного момента времени.
- `timing.cpp` — реализация точного ожидания с комбинированием сна и короткого активного ожидания.

### `src/core/capture`

- `capture_source.h` — общий интерфейс `ICaptureSource`, описание источника и параметры захвата.
- `camera_capture.h` — класс захвата камеры через OpenCV VideoCapture.
- `camera_capture.cpp` — открытие камеры, получение кадров и измерение задержки.
- `screen_capture.h` — класс захвата монитора через Desktop Duplication API/DXGI.
- `screen_capture.cpp` — инициализация D3D11/DXGI, получение текстуры рабочего стола, масштабирование и преобразование кадра.
- `window_capture.h` — класс захвата клиентской области выбранного окна.
- `window_capture.cpp` — захват окна через `PrintWindow` с резервным `BitBlt`, проверка содержимого и преобразование в `cv::Mat`.
- `source_catalog.h` — объявления поиска источников и фабрики объектов захвата.
- `source_catalog.cpp` — перечисляет мониторы DXGI, видимые окна Win32 и камеры DirectShow; создаёт нужный тип `ICaptureSource`.

### `src/core/display`

- `preview_window.h` — оболочка Win32-окна предпросмотра и обработчика сообщений.
- `preview_window.cpp` — регистрация класса окна, создание/закрытие окна, resize и передача событий дисплею.
- `d3d_display.h` — интерфейс Direct3D 11 renderer для загрузки и показа кадров.
- `d3d_display.cpp` — swap chain, текстура, shader-ресурсы, обновление изображения, Present и статистика FPS отрисовки.

### `src/core/hardware`

- `cpu_monitor.h` — типы и интерфейс измерения загрузки CPU процесса.
- `cpu_monitor.cpp` — вычисляет загрузку CPU по времени процесса и количеству логических процессоров.
- `gpu_monitor.h` — типы GPU-телеметрии и интерфейс монитора.
- `gpu_monitor.cpp` — получает имя основной видеокарты и загрузку GPU процесса через PDH/DXGI.
- `memory_monitor.h` — типы и интерфейс измерения памяти процесса.
- `memory_monitor.cpp` — получает working set процесса и процент используемой памяти.
- `system_info.h` — структура общей информации о компьютере и функции форматирования.
- `system_info.cpp` — определяет CPU, GPU, RAM, VRAM, физические и логические ядра для стартового отчёта.

### `src/core/inference`

- `onnxruntime_loader.h` — объявление ручной загрузки ONNX Runtime.
- `onnxruntime_loader.cpp` — загружает `runtimes\directml\onnxruntime.dll`, проверяет API и передаёт его C++-обёртке ONNX Runtime.
- `amd_neural_runner.h` — объявление подключения DirectML execution provider.
- `amd_neural_runner.cpp` — получает DML API из ONNX Runtime и добавляет DirectML provider в параметры сессии. Название историческое: DirectML работает не только с AMD.
- `input_preprocessing.h` — интерфейс подготовки входного изображения YOLO.
- `input_preprocessing.cpp` — преобразует BGR/BGRA, масштабирует изображение и создаёт нормализованный NCHW float-тензор с повторным использованием памяти.
- `class_metadata.h` — объявления работы с метаданными классов модели.
- `class_metadata.cpp` — разбирает словарь `names` из метаданных ONNX и формирует отсортированный список классов.
- `yolo_inference.h` — состояние ONNX-сессии, повторно используемые буферы и интерфейс инференса YOLO.
- `yolo_inference.cpp` — создаёт DirectML-сессию, проверяет формы тензоров, запускает модель, разбирает выход, фильтрует confidence и выполняет NMS.

### `src/core/overlay`

- `overlay.h` — объявление отрисовки результатов детекции.
- `overlay.cpp` — масштабирует координаты, рисует рамки, подписи классов и confidence поверх кадра.

### `src/core/pipeline`

- `pipeline_state.h` — разделяемое потокобезопасное состояние: последний кадр, последние детекции, mutex, condition variable, флаги остановки и атомарная телеметрия.
- `workers.h` — объявления worker-функций захвата и инференса.
- `workers.cpp` — циклы потоков захвата и инференса, ограничение FPS, публикация свежих данных, статистика и передача ошибок.
- `fps_statistics.h` — структуры и интерфейс расчёта статистики FPS.
- `fps_statistics.cpp` — рассчитывает maximum FPS, 1% Low и 0.1% Low по накопленным измерениям.

### `src/core/platform`

- `utf8.h` — объявления преобразования строк и путей UTF-8/UTF-16.
- `utf8.cpp` — преобразования через Windows API для корректной работы с кириллицей в путях.

### `tests`

- `config_tests.cpp` — проверяет чтение конфигурации, обязательные поля, относительные пути, классы и обработку ошибок.
- `fps_statistics_tests.cpp` — проверяет maximum FPS, 1% Low и 0.1% Low.
- `input_preprocessing_tests.cpp` — сравнивает оптимизированную подготовку BGR/BGRA с эталонным OpenCV pipeline и проверяет повторное использование буфера.
- `onnxruntime_loader_tests.cpp` — проверяет загрузку runtime и наличие `DmlExecutionProvider` и `CPUExecutionProvider`.
- `source_catalog_tests.cpp` — проверяет перечисление источников, захват монитора и тестового окна.
- `inference_benchmark.cpp` — измеряет производительность инференса на модели; является benchmark и не входит в стандартный набор из пяти CTest-тестов.

## Потоки и обмен данными

- **Capture worker** получает кадры с заданной частотой, присваивает `frame_id` и заменяет `latest_frame`.
- **Inference worker** ждёт доступный кадр, выполняет YOLO и заменяет `latest_detections`.
- **Runtime/render thread** обрабатывает сообщения окна, берёт свежий кадр и детекции, рисует overlay и вызывает Direct3D Present.
- Доступ к кадрам и детекциям защищён отдельными mutex; завершение координируется атомарными флагами и condition variable.
